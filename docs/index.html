Usage Examples

Simple, powerful, and reliable file locking

🔒 Basic Locking 
const locksmith = require('locksmithx');

(async () => {
  // Acquire a lock on a file
  const release = await locksmith.lock('some/file');
  
  try {
    // Do something while the file is locked
    console.log('File is locked!');
  } finally {
    // Always release the lock
    await release();
  }
})();
⚙️ Advanced Configuration 
const release = await locksmith.lock('path/to/file', {
  stale: 15000,        // 15s before lock is stale
  update: 5000,        // Update every 5s
  retries: 3,          // Retry 3 times
  onCompromised: (err) => {
    console.error('Lock compromised:', err);
  }
});

// ... do work ...
await release();
🔍 Check Lock Status 
// Check if file is locked
const isLocked = await locksmith.check('path/to/file');
console.log(isLocked ? 'Locked' : 'Unlocked');

// Force unlock if needed
await locksmith.unlock('path/to/file');
⚡ Synchronous API 
// Synchronous version
const release = locksmith.lockSync('path/to/file');

try {
  // Do work synchronously
  console.log('File locked synchronously');
} finally {
  release();
}
🔓 Shared Locks 
// Multiple readers can access simultaneously
const release1 = await locksmith.lock('shared/file', { mode: 'shared' });
const release2 = await locksmith.lock('shared/file', { mode: 'shared' });

try {
  // Both can read simultaneously
  console.log('Multiple readers active');
} finally {
  await release1();
  await release2();
}
🔒 Exclusive Locks 
// Only one writer at a time
const release = await locksmith.lock('exclusive/file', { mode: 'exclusive' });

try {
  // Exclusive access - no other processes can lock
  console.log('Exclusive write access');
} finally {
  await release();
}
📊 Analytics & Monitoring 
// Real-time metrics and monitoring
const metrics = locksmith.getMetrics();
console.log('Active locks:', metrics.activeLocks);

// Event-driven monitoring
locksmith.on('lock-acquired', (data) => {
  console.log(`Lock acquired: ${data.file} in ${data.duration}ms`);
});

// Lock tree visualization
const tree = locksmith.getLockTree();
console.log(tree);
🌐 Distributed Locking 
// Redis backend for distributed systems
await locksmith.lock('shared-resource', {
  backend: 'redis',
  redis: { host: 'localhost', port: 6379 }
});

// Consul backend for service discovery
await locksmith.lock('service-lock', {
  backend: 'consul',
  consul: { host: 'localhost', port: 8500 }
});

API Reference

Complete API documentation for all Locksmith methods

lock(file, [options])

Acquires a lock on a file. Returns a Promise resolving to a release function.

file string Path to the file to lock
stale number Milliseconds before lock is considered stale (default: 10000)
update number Milliseconds between lockfile updates (default: stale/2)
retries number|object Number of retries or retry options (default: 0)
onCompromised function Callback if lock is compromised

unlock(file, [options])

Releases a lock on a file. Returns a Promise.

check(file, [options])

Checks if a file is locked and not stale. Returns a Promise resolving to a boolean.

Synchronous Methods

All methods have synchronous counterparts: lockSync(), unlockSync(), checkSync()

Best Practices

Follow these guidelines for optimal file locking

🔒
Always Release Locks
Use try/finally blocks to ensure locks are always released, preventing deadlocks.
⚠️
Handle Compromised Locks
Use the onCompromised callback to handle unexpected lock loss gracefully.
⚙️
Tune Intervals
Adjust stale and update intervals based on your workload and environment.
🔗
Advisory Locking
Remember that Locksmith uses advisory locking - only processes using Locksmith will honor the lock.
